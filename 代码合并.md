# Qwen2.5-VL / Qwen3-VL 合并说明

## 1. 当前支持范围

| 模型族 | 支持参数量 | 数据处理逻辑 | processor |
| --- | --- | --- | --- |
| Qwen3-VL | 4B / 8B | 普通 grounding prompt，视频尺度 `32 * 32`，processor 输入包含 `video_metadata` | 默认使用模型自带 processor |
| Qwen2.5-VL | 3B / 7B | textual timestamp prompt，视频尺度 `28 * 28`，训练时将 GT span 对齐到采样帧时间戳 | 建议显式传 `TencentARC/TimeLens-7B` |

processor 选择规则：

- 传了 `--processor_path`：使用传入的 processor。
- 没传 `--processor_path`：使用 `--model_path` 自带的 processor。

跑 Qwen2.5-VL 3B/7B 时，如果需要论文里的 timestamp 编码，需要保留：

```bash
--processor_path TencentARC/TimeLens-7B
```

评估脚本也遵循同一规则。评估本地 Qwen2.5-VL TimeLens checkpoint 时，建议这样运行：

```bash
model_path="output/TimeLens-Qwen2.5-7B/sft/<your_sft_run_dir>" \
processor_path="TencentARC/TimeLens-7B" \
bash scripts/eval_timelens_bench.sh
```

## 2. 推荐运行流程

整体流程是：

```text
SFT -> filter data -> GRPO
```

脚本矩阵：

| 模型 | SFT | filter | GRPO |
| --- | --- | --- | --- |
| Qwen2.5-VL 3B | `train_scripts/run_sft_qwen25_3b.sh` | `scripts/filter_data/filter_data_qwen25_3b.sh` | `train_scripts/run_grpo_qwen25_3b.sh` |
| Qwen3-VL 4B | `train_scripts/run_sft_qwen3_4b.sh` | `scripts/filter_data/filter_data_qwen3_4b.sh` | `train_scripts/run_grpo_qwen3_4b.sh` |
| Qwen2.5-VL 7B | `train_scripts/run_sft_qwen25_7b.sh` | `scripts/filter_data/filter_data_qwen25_7b.sh` | `train_scripts/run_grpo_qwen25_7b.sh` |
| Qwen3-VL 8B | `train_scripts/run_sft_qwen3_8b.sh` | `scripts/filter_data/filter_data_qwen3_8b.sh` | `train_scripts/run_grpo_qwen3_8b.sh` |

## 3. 关键代码改动

| 文件 | 作用 |
| --- | --- |
| `training/model_family.py` | 统一判断模型族；Qwen3-VL 覆盖 4B/8B，Qwen2.5-VL 覆盖 3B/7B；`resolve_processor_source()` 只负责“显式 processor 优先，否则 model path”。 |
| `training/model_loader.py` | 模型和 config 加载前校验模型族；processor 不再强制校验路径名，避免独立 processor 目录被误拒绝。 |
| `training/params.py` | 新增 `processor_path`，允许模型权重和 processor 分开加载。 |
| `training/data/grounding.py` | 训练数据构造。Qwen2.5-VL 走 timestamp prompt 和 span 对齐；Qwen3-VL 走 `video_metadata` 路径。 |
| `training/data/inference_collator.py` | 过滤推理的数据构造和 batch collate，与训练数据逻辑保持一致。 |
| `training/filter/infer_qwen3_vl_tvg_dataloader_filter_data.py` | 通用 filtering 入口，支持 Qwen2.5-VL 和 Qwen3-VL。 |
| `training/train/train_sft_timelens.py` / `training/train/train_grpo_timelens.py` | 统一 SFT / GRPO 入口，支持 `processor_path`。 |
| `evaluation/eval_dataloader.py` / `evaluation/utils.py` | 评估入口支持 `processor_path`，避免本地 Qwen2.5-VL checkpoint 误用自身 processor。 |

## 4. 注意事项

- Qwen2.5-VL 3B/7B 使用 timestamp 逻辑时，请显式传 `--processor_path TencentARC/TimeLens-7B`。
- Qwen3-VL 4B/8B 默认不需要单独传 processor，除非你确实有独立 processor 目录。
- 所有模型大小都有独立训练入口：3B、4B、7B、8B 分别提供 `sft`、`filter`、`grpo` 脚本。
- `grpo_and_eval` 这类拼接脚本已移除；实际运行时建议先跑 GRPO，确认 checkpoint 后再按通用评估脚本单独评估。
